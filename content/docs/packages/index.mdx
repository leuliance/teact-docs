---
title: Packages
description: npm packages in the Teact monorepo — what to install and when to use each scope.
---

| Package | Install | Use when |
| ------- | ------- | -------- |
| [@teactjs/core](/docs/packages/core) | `bun add @teactjs/core` | You want one import for UI + runtime |
| [@teactjs/react](/docs/packages/react) | `bun add @teactjs/react` | UI-only imports |
| [@teactjs/runtime](/docs/packages/runtime) | `bun add @teactjs/runtime` | Bot engine without UI barrel |
| [@teactjs/ui](/docs/packages/ui) | `bun add @teactjs/ui` | Same as react, different import path |
| [@teactjs/telegram](/docs/packages/telegram) | `bun add @teactjs/telegram` | Production Telegram transport |
| [@teactjs/storage](/docs/packages/storage) | `bun add @teactjs/storage` | Disk / memory / custom persistence |
| [@teactjs/testing](/docs/packages/testing) | `bun add -D @teactjs/testing` | Unit tests |
| [@teactjs/cli](/docs/packages/cli) | `bun add -D @teactjs/cli` | Local `teact` commands |
| [create-teact](/docs/packages/create-teact) | `bun create teact` | New projects |

`@teactjs/renderer` is internal to the reconciler and not published for app use.

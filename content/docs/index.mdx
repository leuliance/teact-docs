Teact lets you ship Telegram bots using the same mental model as React apps: **components**, **hooks**, **routing**, and **JSX**, compiled and run with [Bun](https://bun.sh).

```tsx
function Counter() {
  const [count, setCount] = useState(0);
  return (
    <Message text={`Count: ${count}`}>
      <InlineKeyboard columns={2}>
        <Button text="−1" onClick={() => setCount((c) => c - 1)} />
        <Button text="+1" onClick={() => setCount((c) => c + 1)} />
      </InlineKeyboard>
    </Message>
  );
}
```

## What you can build

- **Messages & keyboards** — inline and reply keyboards with `<Button>`, `<InlineKeyboard>`, `<ReplyKeyboard>`.
- **Routing** — `createRouter` with params, `beforeLoad` guards, and navigation modes (`push`, `replace`, `stack`, `dismiss`).
- **Flows** — multi-step **conversations** and schema-driven **forms**.
- **Platform features** — **streaming** text, **i18n**, **payments** (`useInvoice`), **auth** plugins, **session** storage, and **events** via `useOn`.
- **Deploy anywhere** — long-running polling _or_ serverless via a web-standard `bot.fetch(request)` handler.

## Packages at a glance

| Package | Role |
| -------- | ---- |
| `@teactjs/core` | The engine — `createBot`, router, hooks, plugin host, reconciler, and re-exported React |
| `@teactjs/ui` | Components — `Message`, keyboards, media, polls, formatting |
| `@teactjs/telegram` | grammY-based adapter (polling & webhook) + `conversationsPlugin` / `streamPlugin` |
| `@teactjs/plugin-sdk` | `definePlugin` — author your own plugins |
| `@teactjs/storage` | Persistent storage plugin (file/memory drivers; bring your own for KV/Redis) |
| `@teactjs/testing` | `MockAdapter`, `renderBot` |
| `@teactjs/cli` / `create-teact` | Dev server, build, deploy, and scaffolding |

<Callout type="info">
  The 0.2.0 rewrite folded the old `@teactjs/react`, `@teactjs/runtime`, and `@teactjs/renderer`
  packages into `@teactjs/core`. Import **components** from `@teactjs/ui` and **everything else**
  from `@teactjs/core`.
</Callout>
